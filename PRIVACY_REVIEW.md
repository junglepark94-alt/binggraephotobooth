# 개인정보 검토 안내 (정보혁신팀용)

빙그레 네컷 — 사내 이벤트용 웹 포토부스. 이 문서는 **"촬영한 사진이 외부로 나가는가"** 를
코드로 직접 확인하실 수 있도록 검토 지점을 정리한 것입니다.

> 전체 구조는 [PROJECT_SUMMARY.md](./PROJECT_SUMMARY.md), 개발 가이드는 [CLAUDE.md](./CLAUDE.md) 참고.

---

## 결론

**촬영·합성·꾸미기·저장이 전부 사용자 브라우저 안에서만 처리되며, 사진은 서버로 전송되지 않습니다.**
사진을 서버로 보내는 코드는 존재하지 않습니다.

서버(Railway)는 **정적 파일 서빙과 SSR(HTML 렌더링)만** 담당합니다. 데이터베이스가 없고,
사진을 받는 엔드포인트도 없습니다.

## 1. 사진이 만들어지고 사라지는 경로

| 단계 | 처리 위치 | 파일 |
| --- | --- | --- |
| 카메라 촬영 (4컷) | 브라우저 `getUserMedia` → `<canvas>` | `src/screens/ShootScreen.tsx` |
| 프레임 합성 | 브라우저 `<canvas>` | `src/lib/photobooth.ts` |
| 스티커 꾸미기 | 브라우저 `<canvas>` | `src/screens/ResultScreen.tsx` |
| 저장 | 브라우저 다운로드 (`<a download>`) 또는 이미지 길게 눌러 저장 | `src/screens/ResultScreen.tsx` |
| 공유 | OS 공유 시트 (Web Share API) — 앱이 아닌 **사용자가** 대상 선택 | `src/screens/ResultScreen.tsx` |

사진 데이터는 브라우저 메모리와 캔버스에만 존재하고, 화면을 벗어나면 사라집니다.
서버·로컬스토리지·쿠키 어디에도 저장하지 않습니다.

## 2. 직접 확인하실 수 있는 검증 방법

아래 명령으로 "서버 통신 코드가 없다"를 직접 확인하실 수 있습니다.

```bash
# 1) 서버 함수(RPC) 정의가 하나도 없음 — 결과 없음이 정상
git grep -n "createServerFn" -- src/

# 2) 클라이언트에서 나가는 네트워크 호출 — 아래 1건 외에 없음
git grep -nE "\bfetch\(|XMLHttpRequest|axios" -- src/

# 3) 사진 업로드 기능이 제거된 커밋
git show 14f738e --stat
```

### 검토 시 오해하기 쉬운 지점 두 가지

**(1) `src/screens/ResultScreen.tsx`의 `fetch()`** — 네트워크 호출이 아닙니다.

```ts
const blob = await (await fetch(url)).blob();
```

여기서 `url`은 `exportImage()`가 만든 **`data:image/png;base64,...` 형식의 문자열**입니다.
브라우저에서 data URL을 Blob 객체로 바꾸는 표준 관용구이며, 외부로 나가는 요청이 아닙니다.
바로 다음 줄에서 그 Blob을 Web Share API에 넘겨 **OS 공유 시트**를 띄웁니다 —
공유 대상(카톡·메일 등)은 앱이 아니라 사용자가 그 자리에서 직접 고릅니다.

**(2) `src/server.ts`의 `handler.fetch(request)`** — 서버 측 SSR 진입점입니다.
들어온 요청을 받아 HTML을 렌더링하는 표준 핸들러이고, 사진과 무관합니다.

## 3. 외부로 나가는 통신 (전부)

앱이 자기 자산 외에 접촉하는 곳은 아래 두 곳뿐입니다.

| 대상 | 용도 | 사진 전송 |
| --- | --- | --- |
| `fonts.googleapis.com` / `fonts.gstatic.com` | 웹폰트 로딩 (`src/routes/__root.tsx`) | 없음 |
| `open.kakao.com` | 이벤트 응모 — **사용자가 버튼을 눌렀을 때만** 새 탭으로 이동 (`src/data/event.ts`) | 없음 |

카카오톡 링크는 단순 페이지 이동이며, 앱이 사진이나 개인정보를 함께 보내지 않습니다.
**응모자는 카톡 오픈채팅방에 본인이 직접 사진을 올립니다** — 이 앱을 거치지 않습니다.

**애널리틱스·트래킹 도구는 설치되어 있지 않습니다.** (Google Analytics, Sentry 등 없음)

## 4. 이전 버전에서 제거된 기능

초기 버전에는 촬영한 사진을 사내 게시판("광장 게시판")에 올려 공유하는 기능이 있었고,
서버(Redis)에 사진이 저장됐습니다. **개인정보 보호를 위해 이 기능 전체를 제거했습니다.**

- 제거 커밋: **`14f738e`** — "사진 게시판 업로드 제거 → 이벤트 응모(카톡) 플로우로 전환"
- 삭제된 것: 업로드/조회/좋아요 서버 함수(`src/lib/plaza.ts`), 관리자 페이지(`src/routes/admin.tsx`),
  게시판 화면(`src/screens/PlazaBoard.tsx`), 업로드용 이미지 축소 함수
- 서버에 남아 있던 기존 사진은 **관리자 페이지에서 전량 삭제 후 배포**했습니다.
- 사진 공유는 게시판 대신 **카카오톡 오픈채팅 응모**로 대체됐습니다.

`git log`로 제거 이력을 확인하실 수 있습니다.

## 5. 자격증명·비밀정보

현재 소스와 전체 커밋 히스토리에 **하드코딩된 비밀번호·API 키·토큰이 없습니다.**
`.env` 파일도 사용하지 않습니다. (과거 관리자 기능도 비밀번호를 환경변수로만 읽었고,
그 코드마저 위 커밋에서 삭제됐습니다.)

배포에 필요한 환경변수는 없으며, 서버는 정적 파일 + SSR만 수행합니다.

## 6. 권한 요청

앱이 사용자에게 요구하는 권한은 **카메라 하나**입니다.

- 촬영 화면에서 사용자가 "허용하고 시작하기"를 누른 뒤에만 요청합니다 (`src/screens/ShootScreen.tsx`)
- 카메라 스트림은 촬영 화면을 벗어날 때 즉시 종료합니다
- 마이크·위치·연락처 등 다른 권한은 요청하지 않습니다
